# 关于C的补充复习
### 隐式转换和强制转换
从取值范围跨度的角度看，C的各种数据类型从大到小排列为：
double>float>longlong>long>int>short>char  

不同的类型的数据在进行运算、赋值等操作时会对数据进行隐式转换，具体表现为：
1. 范围小的数据类型向范围大的数据类型转换
2. char、short类型的数据会转换为int

强制转换一般是对范围大的数据类型手动向范围小的数据类型转换，注意范围的大小可能导致数据错误

### 数组

1. #### 数组名的解释：
   数组名（arr）的本质是代表整块数组内存的「只读标识符」，不是变量（运行时不占用额外内存，无法修改，如arr++编译报错）；绝大多数场景（取值、传参、算术运算）下，编译器会将其隐式降级为「指向数组首元素的常量指针」（等价于&arr[0]）,这里的降级最明显的一点就是用sizeof计算出来的是数组大小，arr不能改变指向，另一个表现就是后面谈到的。而在这里就需要把arr和&arr区别开：&arr本质上是一个数组指针,它直接指向arr数组的首地址。当然从这个角度看arr和&arr数值上是相同的，但区别在于：arr+1,地址偏移一个元素；&arr+1,地址偏移整个数组大小的地址。

   理解上述部分后，我们就能明白arr[1]这样的调取数组内容的书写逻辑。arr经过降级变为常量指针，便可以用解释调取地址数据，*(arr+1)与arr[1]是同一个数据调用的不同书写形式。对于&arr[1]这样的调用数据地址，就是直接指向1号数据的地址。
2. #### 多维数组的解释：
   C的多维数组本质上是一个一维数组，通过指定维度大小对连续空间分类成为多维数组，此举可以解释为什么多维数组定义初始化时，有类似arr[][3]={1,2,3,4,5,6}的方式，因为系统只要知道低维度的大小，就能自动计算出高纬度的大小。

   此处多维数组的名称arr，也是能直接调用隐式转换为指针，不过注意：arr[0]和arr[1]也是当前行的一个一维数组标识符，这就跟上面的一维数组名解释一样，它也能直接调用隐式转换为指针，或者说，它的用法是跟一维数组是一样的。


### 指针补充

任何数据类型定义下的指针变量，本身的空间是大小一样的，数据类型实际上是规定了两个规律：
1. 通过该指针解引用存储的数据，要一次性取多大的字节量 
2. 规定指针算数时，对应指向地址变化多大的偏移量(步长是多少字节)

#### 指针的运算：
1. 与整数的加减操作
2. 指向同一连续内存(如数组)的两个指针：  
   a.做差，结果是两个地址之间的元素个数  
   b.=/!=：判断两个指针是否指向「同一地址」  
   c.</>/<=/>=：判断指针在内存中的 “先后顺序”    
   d.空指针（NULL）仅能和其他指针做==/!=比较  
3. = 等号赋值操作，是在进行地址的复制

#### 函数指针：

函数指针：指向函数在内存中执行入口地址的指针  
普通定义和调用：
```
返回值类型 (*函数指针名)(形参数据类型1...);
```
```
函数指针名(实参1...);
```
数组定义和调用:
```
返回值类型 (*函数指针名[数组大小])(形参数据类型1...);
```
```
函数指针名[标号](实参1...);
```

### 结构体内存补齐
自身大小：每个成员的类型大小（如char=1、int=4、double=8）；  
默认对齐系数：编译器默认的对齐单位（VS 默认 8 字节，GCC 默认取「结构体最大成员大小」，可通过#pragma pack修改）；  
有效对齐值：每个成员的「自身大小」和「默认对齐系数」中的较小值（成员偏移量、结构体总大小都必须是有效对齐值的整数倍）。  

规则 1：单个成员的起始偏移量  
每个成员的起始偏移量必须是「自身有效对齐值」的整数倍（若前一个成员结束位置不满足，编译器会自动填充空白字节）。   
规则 2：结构体的总大小  
结构体的总大小必须是「结构体最大成员的有效对齐值」的整数倍（若最后一个成员结束位置不满足，编译器会在末尾填充空白字节）。  
规则 3：嵌套结构体的对齐  
嵌套结构体的起始偏移量必须是「嵌套结构体内部最大成员的有效对齐值」的整数倍；  
嵌套结构体的总大小也需按「内部最大成员的有效对齐值」对齐。

### 动态内存分配
1. malloc(size);  
   在堆区申请一块连续的、大小为size字节的内存块
   申请成功返回内存块的起始地址；申请失败（如内存不足）返回NULL。

2. calloc(num,size);
   在堆区申请num个、每个大小为size字节的连续内存（总大小num*size），会自动把申请到的所有字节初始化为 0。
 
3. realloc(ptr,new_size);
   调整ptr指向的已申请内存的大小为new_size字节，是实现动态数组扩容 / 缩容的核心函数。

上述三个一般需要前面加上强制类型转换，防止申请失败产生NULL导致数据错误。

4. free(ptr);  
释放ptr指向的、由malloc/calloc/realloc申请的堆内存，将内存归还给系统，避免内存泄漏
