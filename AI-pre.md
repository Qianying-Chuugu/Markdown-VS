# 人工智能导论
## 搜索技术
### 1.图搜索算法

OPEN表：存放待扩展的节点  
CLOSED表：存放已扩展的节点  

① 把起始节点S放到OPEN未扩展节点表中。  
② CLOSED已扩展节点表，其初始为空表。  
③ LOOP：若OPEN表是空表，则搜索失败退出。  
④ 选择OPEN表上的第一个节点，把它从OPEN表移出，并放进CLOSED表中。记这一个节点为节点 n 。    
⑤ 若n为目标节点，则有解并成功退出，所求的解是G中沿着指针从 n 到 S 这条路径。  
⑥ 否则，扩展节点n ，生成不是n祖先的那些后继节点的集合 M。把M的成员作为n的后继节点加到图G中。  
⑦ 对既不在OPEN表中又不在CLOSED表中的M成员设置一个通向n的指针，并依次加进OPEN表。对已经在OPEN或CLOSED中的M的成员，确定是否需要更改通到父节点 n 的指针方向(选择返回起始节点S代价更小的路径)  
⑧ 按某一任意方式或按某个评价函数，重排OPEN表(这决定了其他图搜索算法)。  
⑨ GO LOOP。  

解是在搜索树G中反向追踪从目标节点到起始节点的指针的一条路径

### 2.无信息（盲目）搜索
1. 广度优先搜索中采用先进先出队列FIFO排列OPEN表
   
从4种度量来评价广度优先搜索：  
   完备性——总能找到一个解  
   最优性——如果每步扩展的消耗相同时，广度优先搜索能找到最优解  
   指数级的时间消耗  
   内存消耗是比执行时间消耗更大的问题  
   
2. 深度优先搜索中采用后进先出队列LIFO排列OPEN表  

唯一优势：内存需求极低，DFS 仅需保存当前正在遍历的单条路径节点，不需要存储整层节点

不具备完备性：如果搜索树存在无限深度的无效分支，DFS 会深陷其中、无法回溯，即使问题有解，也可能永远找不到解。(深度有限搜索，但容易无解)

不具备最优性DFS： 找到的第一个目标节点，无法保证是路径最短、代价最低的最优解。

迭代加深深度优先搜索:从零开始逐层递增深度限制，反复调用深度有限搜索,具有完备性和等代价下最优性。

### 3.启发式搜索(评价函数)
1. 贪婪最佳优先搜索算法 
 
f(n)=h(n)，h(n)计算每个节点距离目标节点的最低估计代价，OPEN表节点的排列优先较低代价的节点  
空间占用少，效率高，不完备，非最优  

2. A*搜索    

f(n)=g(n)+h(n),g(n)计算每个节点真实的起始节点到当前状态的代价，h(n)仍是估计代价，OPEN表节点的排列优先较低代价的节点  

只有当h(n)的计算规则永远保证比真实的未来代价小，才能保证搜索结果是完备的、最优的，且任何一个可采纳h(n)，A*算法效率最优，同时h(n)越接近真实未来代价效率越高

### 4.博弈搜索
1. 极大极小值算法  

双方轮流走棋：  
一方 MAX（极大者）：追求局面评估值最大  
一方 MIN（极小者）：追求局面评估值最小  

博弈树分层交替：  
MAX 层 → 取所有子节点的最大值  
MIN 层 → 取所有子节点的最小值  
从叶子节点向上倒推选值
最底层叶子节点：直接给出局势评估分数    

能预判对手策略，得到博弈最优决策
博弈树层数一多，节点暴增，计算量爆炸、搜索效率低  

2. α-β 剪枝算法  

还未发生比较时：  
MAX层记录当前值α=−∞  
MIN层记录当前值β=+∞  
  
MAX和MIN层叠取值，比较出一支，就回传父节点，比较剩余的支时，只要α≥β，则当前支不影响父节点取值，可以直接剪支

### 5.局部搜索
不关心路径的搜索算法就是局部搜索算法  
1. 爬山法搜索

向值增加的方向持续移动（登高过程），如果没有比当前状态更好的子状态（到达顶峰），则算法结束，但是极易陷入局部最优值

2. 模拟退火搜索

引入温度 T 控制搜索  
产生新解：  
新解更优：直接接受  
新解更差：按一定概率选择性接受  
随着迭代，温度慢慢降低  
高温阶段：大概率接受差解 → 大范围探索  
低温阶段：几乎只接受优解 → 逐步收敛  
恶化解的接受概率：P=e**−(ΔE/T)​

模拟退火算法的优点:质量高；初值鲁棒性强；简单、通用、易实现。  
模拟退火算法的缺点:由于要求较高的初始温度、较慢的降温速率、较低的终止温度，以及各温度下足够多次的抽样，因此优化过程较长。

3. 遗传算法

A编码设计：把实际问题的解，转换成算法可处理的染色体编码（最常用二进制编码、实数编码）  

B初始化种群：随机生成 N 个个体，构成初始种群，同时设定迭代次数、交叉 / 变异概率等核心参数  

C适应度计算：为种群内每个个体，计算对应的适应度值 

D选择操作：遵循 “适者生存”，按适应度高低筛选优秀个体作为父代，适应度越高，被选中繁殖的概率越大  

E交叉操作：选中的父代个体交换部分基因片段(单点交叉、多点交叉)，生成新一代子代个体，是种群进化的核心动力  
F变异操作：以极低概率随机改变个体的某个基因(位翻转)，维持种群多样性，避免算法陷入局部最优  

G终止判断：若达到最大迭代次数、或适应度达标，算法终止，输出种群中最优个体；否则返回步骤 3，用新一代种群继续迭代

模式是某些位置上的数字尚未确定的一个状态子串  
模式定理：适应值高于群体平均值的模式在下一代的代表串数目将会增加，而适应值低于群体平均值的模式在下一代的代表串数目将会减少。

## 逻辑推理
### 1.确定性推理

1. 自然演绎推理

Horn子句是一阶逻辑中专门用于高效推理的特殊子句(等价于一个蕴含式)：  
子句：  多个逻辑文字的析取（或运算）  
限制：最多只能有 1 个正文字(未取否定的命题)，称为头(推理结论)，其他的负文字称为体(推理条件) 

前向链接算法：  
事实集合向规则集合匹配前提，如果成功则进行事实和变量的置换{变量/事实}，得到的结果作为事实纳入事实集合，直至获得结论。

后向链接算法：(效率更高) 
新增一个栈，初始栈中存结论，弹栈，把规则集合中该结论的前提纳入栈，后续循环弹栈；如果弹出的是前提，则与事实集合中向匹配，存在相应的事实则消去该前提；循环往复，当栈空时则证明结论成立。

2. 归结演绎推理——消解法

谓词公式转子句集的步骤：  
A.书写初始条件和结论取反的谓词公式  
B.  P→Q  ⇔ ﹁ P∨Q   
   P↔Q  ⇔  (P∧Q)∨(﹁P∧﹁Q)  
C.用德摩根律、量词否定律，把否定符号移到原子谓词前：  
¬∀xP(x)⇒∃x¬P(x)
¬∃xP(x)⇒∀x¬P(x)  
D.给所有量词的约束变量重命名，确保每个变量仅被一个量词约束，避免变量名冲突。  
E.若存在量词不在任何全称量词的辖域内：用一个全新的常量符号替换该变量  
若存在量词在 k 个全称量词的辖域内：用一个全新的 k 元斯科伦函数替换该变量，参数是这 k 个全称量词的变量
F.把所有全称量词∀移到公式的最左侧，得到前束范式  
G.用分配律展开，把母式转换成「子句的合取」形式  
H.省掉全称量词,所有合取的子句拆分，得到最终的子句集 S
I.对子句集中的某些变量重新命名，使任意两个子句中不出现相同的变量名

正负原子公式合并 (消解)得到消解式，并将消解式(新子句)加入S中，并反复执行这一过程，最后集合中出现空子句，说明整个子句集是不可满足的，即恒为假，反证原结论成立

### 3.不确定性推理
从不确定性的初始证据（即已知事实）出发，运用不确定性的知识（或规则），推出具有一定程度的不确定性但却是合理或近乎合理的结论   
标准规则形式：IF E THEN H (CF(H,E))  
E：观测证据、已知前提  
H：待推导的结论、假设  
CF(H,E)：可信度因子，量化「证据 E 出现时，结论 H 成立的不确定置信程度」  

1. 全联合概率分布推理  
  
CF(H,E)是事件E发生时H发生的条件概率P(H∣E) 

2. Bayes(贝叶斯)推理  

P(H∣E)=[P(E∣H)​/P(E)]⋅P(H)

P(H)：先验概率  
未观测到任何证据时，对假设H成立的初始基础置信度。  
P(H∣E)：后验概率  
确认证据E为真后，更新得到的、假设H的最新可信概率。  
P(E∣H)/P(E)​：  
修正因子（似然比）承载新证据的全部信息，用来倍率调整先验概率。

随着新证据的不断获得，从证据少时的后验概率推出证据多时的后验概率，且每一步都是把上一步的后验概率视为新证据到来时的先验概率

3. 贝叶斯网络推理

假设节点E直接影响到节点H，即E→H，则用从E指向H的箭头建立结点E到结点H的有向弧(E,H)，权值(即连接强度)用条件概率P(H|E)来表示。贝叶斯网络主要用来描述随机变量之间的条件依赖，圆圈表示随机变量，箭头表示条件依赖。

求全联合概率分布：  
网络内每一个节点，在它直接父节点条件下的局部条件概率，全部累乘得到

推理过程：  
         P(X|e)= aP(X,e)= a∑yP(X,e,y)  
X为待查询变量,e为观察到的证据,y是X其他有关未知隐变量，取定所有可能的取值并计算概率之和，a归一化系数，保证最终概率总和为1。
一般的，要求X|e发生和未发生的概率，相加再分别求占比才有真实的概率，这个过程就是a的归一化。

单连通网络(无环)下推理的复杂度呈现线性关系，多联通网络一般做近似算法

### 4. 模糊推理
A={<x, A(x)>| x∈U，A(x)∈[0, 1]}，则A称为集合U的一个模糊子集。A(x)称为x对A的隶属函数，或隶属度、一致性测度

模糊集合的性质体现在隶属函数上：  
A(x)恒为0，则A为空集   
A是B的子集，则A(x)恒小于等于B(x)  
A的补集，A(x)=1-A(x)  
A与B的并集，N(x)取A(x)B(x)较大的那个  
交集则是取较小的那个  
A-B差集，取A(x)和1-B(x)较小的那个  
模糊集合满足交换、幂等、分配、狄摩根以及交并子集性质  

定义在模糊集合上的关系就是模糊关系，一个模糊关系R也有自己的一套表征关系的隶属函数(一个方阵)，这个隶属函数的具体定义有三种：  
1. Rm  
Rm的边(a,b)关系的隶属值，为元素a和b隶属值的较小值min，a隶属值的补au,min和au取得的较大值
2. Rc  
Rm的边(a,b)关系的隶属值，为元素a和b隶属值的较小值min
3. Rg  
Rg的边(a,b)关系的隶属值，当a的隶属值小于等于b时取1，否则取b的隶属值  

模糊关系的合成：  
R1隶属值方阵A，R2隶属值方阵B，类似于矩阵的乘积，新隶属值方阵C的每个值如此计算：(i,j)属于C，是A的第i行每个值与B的第j列每个值对应比较取较小值后，整体结果取得的最大值

模糊关系的推理：  
模糊命题 x is A (CF)   CF是命题可信度  
模糊股则 IF E  THEN H (CF,λ)   
λ是匹配阈值：只有证据和前提的匹配度大于等于该阈值，这条规则才会被启用、触发

两个模糊集或模糊概念的相似程度称为匹配度，通常有以下三种方式表征(模糊集合A,B是同一个论域U上的)：  

1. 贴近度 
    
(A, B)= [A∙B+(1-A⊙B)] /2  
前者是内积，表示两个模糊集合，每个隶属度先取较小再整体取最大；后者是是外积，表示两个模糊集合，每个隶属度先取较大再整体取最小  

2. 海明距离

A,B的海明距离X，是所有对应隶属度|A-B|后求总和，除以集合元素数N。1-X能作为匹配度

3. 相似度

最大最小法：  
所有对应隶属度求较小后求总和得MIN，所有对应隶属度求较大后求总和得MAX，MIN/MAX即为所求

推理规则:  
1. 模糊假言推理

IF   x  is  F   THEN   y  is  G  
由F'推G'，F'与F匹配，需要求得F到G的模糊关系R，实际上G'=F'*R

2. 模糊拒取式推理

IF   x  is  F   THEN   y  is  G  
由G'推F'，G'与G匹配，需要求得F到G的模糊关系R，实际上F'=R*G'

3. 模糊假言三段论推理  

设F、G、H分别是U、V、W上的3个模糊集，且由知识  
        IF  x  is  F  THEN  y  is  G  (R1)
        IF  y  is  G  THEN  z  is  H  (R2)
则可推出:      IF  x  is  F  THEN  z  is  H(R3)  
R3 = R1οR2,一般这样求得的R3更为精确真实

### 2. 分类算法
1. 决策树

决策树是从根到叶子节点延申分类的树状判别方法，其中叶子节点是最终的分类结果，其他节点都是属性，每个节点的出度是该节点属性的一种选择。

决策树的生成赖于属性判别的先后顺序，良好的顺序会使树的深度较小，判别效率高。  
信息熵：描述一组信息类别的混乱程度  
H(X)=-Plog2P求各类总和  
P是该分类下一种类信息占总信息的比重  

条件信息熵：  
是在该属性条件下分类的信息，仍然按照最终目的分类计算的信息熵A、B、C...,然后按照分别的比例加权求和

先计算最终目的分类下的信息熵A，然后依据该信息熵计算不同属性下再分类后的条件信息熵B，计算所有的A-B，那种熵的下降程度大就下一步按照哪个属性生成决策树。

根节点信息熵最大，叶子节点信息熵归零  

决策树剪枝：防止过拟合，提高泛化能力  
A预剪枝：规定深度、某个节点下的属性情况只有一种、该节点分类下的实例过少等情况下，构建决策树时直接剪枝，该节点作为叶子节点  
B后剪枝：生成完整的决策树后，对那些置信度不够的结点子树用叶子结点来代替，该叶子的类标号用该结点子树中最频繁的类标记



2. 朴素贝叶斯分类  

由特征反向推分类成立的概率，即贝叶斯概率的思想，所有特征之间假设是相互独立、互不影响的。  
极大似然估计就是利用这种思想分类的算法。  
m估计：  防止一个属性在一组样本中出现概率为0
根据样本计算属性概率P=N/Nc  
m估计添加了m个虚构样本，  
计算对应属性概率P=(N+mp)/(Nc+m)
其中p=1/C,C是该属性所有可能的取值个数

3. 集体学习

集体学习是选择多个分类函数，将它们对新实例的分类预测进行合成，然后再输出结果，以此降低预测错误概率。  
集体学习 = 把一堆弱学习器组合成强学习器

A.Bagging学习算法——随机森林  
算法：对所有数据进行有放回抽取，得出N个独立集合，并行训练出N个基分类器，最终使用时把N个分类器的结果类似投票的方式选择输出

随机森林就是N个决策树组成的，  
每棵树的按照如下规则生成：  
a. 给定一个训练样本集，数量为N，使用有放回采样到N个样本(会采样到重复的样本)，构成一个新的训练集。  
b. 从总量为M的特征向量中，随机选择m个特征（m<M,无放回的选择），其中m可以等于m=M的开平方。（在整个森林的生长过程中， m的值一般维持不变）  
c. 使用一般决策树的构建方法，得到一棵决策树。  
d. 重复这样的过程H次，就得到了H棵决策树。形成一个随机森林  

其中选择的特征数m的取值决定了森林分类效果的好坏，对于m的各种取值，所对应的袋外错误率(OOD ERROE)越小越好:  
每个决策树的构建训练样本，总有大约原始样本1/3的样本未被参与训练，对于每个原始样本，计算它在所有自己未参与训练的决策树中的分类，投票得出分类结果。最终分类结果错误的样本占总样本的比率就是袋外错误率。袋外错误率，是随机森林泛化误差的无偏估计

不容易陷入过拟合、抗噪声能力强、无需提前降维特征量、自带内部无偏误差估计（OOB 袋外误差）、并行训练模型速度快

B.Boosting学习算法——AdaBoost  
算法：串行依次训练分类器，一轮训练完成，主动提高预测错误样本的权重；基分类器识别准确率越高，最终整体占比权重越大；最后使用时把所有分类器的结果进行加权求和选择结果输出  

输入新样本，所有弱模型各自给出类别预测  
把预测为同一类别的所有模型权重求和  
权重总和最大的类别，就是 AdaBoost 最终输出结果  

4. 支持向量机（SVM）

结构风险最小化(SRM)：
传统的模型训练，训练集样本有限，导致模型容易经验风险最小化，严格按照训练集规律，对未知数据处理能力弱。    
  
模型复杂度(VC维)越低，经验风险越高，置信范围越低    
模型复杂度越高，经验风险越低，置信范围越高   
SRM要求训练时不只照顾训练集正确率，同时限制模型复杂度VC，保持对未知数据的泛化能力,找到最小化经验风险和最小化复杂度的平衡点   

SVM支持向量机：  
所有的样本能在特征维度上找到位置，不同的集群就是不同的分类。以二维空间为例，两个分类集群能找到一个分界直线，这样判别新样本分类时直接根据与直线的位置就能确定分类，这条线叫分界线，往往是两个集群的支持向量确定的直线形成的带状区域(硬间隔)的中央线。  
SVM 的终极目标：最大化这个间隔的宽度，就能让结构风险最小化

集群的划分有时候不能用直线区分：  
A.非线性问题  
通过非线性变换将它转化为某个高维空间中的线性问题，在这个高维空间中寻找最优分类面。  

为了解决升维造成的运算爆炸和过拟合，使用核函数进行低维运算，结果与高维运算相同。进行满足Mercer 条件（对称、半正定）的函数，都可以作为合法核函数

B.离群点问题  
部分样本点因为噪声问题会严重偏离分类集群，而如果仍采用硬间隔，会导致过拟合问题。  

采用松弛因子，分界线允许部分样本偏离集群区域(越大容错越大)  
采用惩罚因子，设定对错误分类的容忍程度(越大容忍越低)  
二者平衡构成软间隔SVM支持向量机

### 4. 聚类算法

1. K均值聚类

a.随机选择K个随机的点（称为聚类中心）  
b.对与数据集中的每个数据点，按照距离K个中心点的距离，将其与距离最近的中心点关联起来，与同一中心点关联的所有点聚成一类；  
c.计算每一组的均值，将该组所关联的中心点移动到平均值的位置；  
d.重复执行2-3步，直至中心点不再变化

2. 密度聚类

a.通过检查数据集中每个点的Eps邻域来搜索簇，如果点p的Eps邻域包含的点多于MinPts个，则创建一个以p为核心对象的簇  
b.迭代地聚集从这些核心对象直接密度可达的对象，这个过程可能涉及一些密度可达簇的合并；  
c.当没有新的点添加到任何簇时，该过程结束

3. 层次聚类   

a.算法最初将每个对象做为一个簇，然后这些簇根据某些准则被一步步的合并，使用单链接方法； 
b.两个簇间的相似度由这两个不同簇中距离最近的数据点对的相似度来确定。此外当两个簇最近距离超过用户给定的阈值时聚类过程就会终止；  
c.聚类的合并过程反复进行直到所有的对象最终满足簇数据。
