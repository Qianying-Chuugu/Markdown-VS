# 3.12 基本语法与运算符
#### 字面量：程序代码里直接写入和利用的数据(值)
1. 整数   
2. 小数 
3. 字符串(直接用“”包含的数据) 
4. 字符(直接用‘’包含的单个数据) 
5. 布尔(true/false) 
6. 空(null)
#### 数据类型：
- 整数类：byte(1字节)、short(2字节)、int(4字节)、long(8字节,赋值结尾为L)
- 小数(浮点)类：float(4字节,赋值结尾为F)、double(8字节)
- 字符类：char(2字节,只能储存单个数据)
- 布尔类：boolean(1字节)
#### 键盘输入：
1. 需要导包： import java.util.Scanner;
2. 创建Scanner对象：Scanner sc = new Scanner(System.in);
3. 调用：
- sc.nextInt() 读入整数         
- sc.nextDouble() 读入小数
- sc.next() 读入字符串(读到空格为止)     
- sc.nextLine() 读入一整行(读到换行符\n为止)
#### 文本输出：
System.out.println();   
执行打印文本并换行，若不想换行则去除ln  
IDEA里直接写sout能自动识别补全为该语句
#### 数据类型转换：
1. 隐式转换  
不同类型的数据进行算数运算，默认先将byte、short类数据转化为int类，之后把所有数据向原数据中取值范围最大的类型转化。
2. 强制转换
#### 字符串拼接：
  任何数据 + 字符串 或者 字符串 + 任何数据 都是直接把两者按前后顺序直接拼接起来。
#### 逻辑运算符：
！、&&、||、&、|  
其中双写的为顺次运算，左边能定下结果则不再运算右边；单写的为顺时运算，两个条件都要运算完毕
#### 三元运算符：
关系表达式？表达式1:表达式2；     
关系表达式成立则执行1，否则执行2


# 3.13 基本结构与数组
#### if判断：
判断条件小括号里只能写逻辑表达式或布尔类型变量
#### switch选择：
C只支持整形或枚举，而java支持整形、字符串和枚举等，范围更广，但不支持小数或long  
基本用法和特性与C相同，java的switch能同时返回值：  
```
int a = switch(表达式){
case 常量1,常量2,....... -> { 
..................................;
yield 返回值；}
};
```
1. 用->代替：,省略手动添加break，自动防穿透
2. ->后接一行指令则不需要{}
3. 同一个case能同时判断多个常量情况，不同情况用逗号隔开
4. ->后接的语句块中，yield表征switch接受的数据，单行指令则不用特意表征，自动接收该行所有数据
5. 基本用法最后不用加；，这种写法最后需要加；
#### for循环：基本用法同C
#### while循环：基本用法同C
#### do-while循环：基本用法同C
#### break和continue: 基本用法同C
#### 数组静态初始化：
完整格式：数据类型[] 数组名 = new 数据类型[]{元素1，元素2........};  
简写格式：数据类型[] 数组名 = {元素1，元素2.........};
#### 数组元素的获取与修改：基本用法同C
#### 数组的长度：数组arr的长度是arr.length
#### 数组的遍历：IDEA里写 数组名.fori 能自动识别补全为for遍历语句
#### 数组动态初始化：
没有初始化的数组，数组的每个空间都会根据数组定义类型默认赋值，具体为：  
- 整形：0   
- 浮点型：0.0   
- 布尔型：false   
- 字符型：\u0000,即空字符 
- 引用型：null    
格式：：数据类型[] 数组名 = new 数据类型[空间大小]；  
#### 二维数组：
1. 静态初始化简写：数据类型[][] 数组名={{1,2},{3,4}};
2. 动态初始化：数据类型[][] 数组名 = new 数据类型[空间大小][空间大小]；
3. ***需要注意的是***：java的多维数组实际上是在数组套数组，除了最低维的标号在指示存储数据的地址，高维的标号都在指示比他低一维的各数组地址。因此在利用arr.length时，指示的是最高维的数组存储地址长度，而非精确到所有存储数据的地址长度，如需调用其他维度数组的地址长度，应该用arr[1].length等。
4. 显然，多维数组的数据地址存储不连续，实际上数组内部各维的元素长度也是可以任意设置的。***动态初始化可以写为：***  
数据类型[][] 数组名 = new 数据类型[空间大小][]  
然后再分别规定各数组长度：arr[0]=new int[2];
#### 随机数生成(Random)：
需要导包：import java.util.Random;  
需要创建随机对象：Random r=new Random();  
生成随机数：  
- r.nextInt(n)    生成0~n-1的整数，不指定n则生成任意的int范围的数
- r.nextDouble()   生成0.0~1.0的小数
- r.nextBoolean() 生成true/false
- 生成任意范围的整数[min,max]:  
r.nextInt(max-min+1) + min
# 3.14 方法和存储原理
### 静态方法：
1. 定义(必须在类下定义)   
  ```
  public static 返回值类型 函数名(定义各形参){
    其他语句；
    return 返回值；}
  ```
在与main同一个类的方法，基本的执行原理与性质同C的函数(直接用函数名传入实参)，不同的类下的方法，还需要 类名.函数名(实参) 这样调用。 

静态方法的静态是指的方法的通用性，静态的方法的执行不依赖于对象，而普通的方法必须要先通过它的类定义对象后，通过对象直接使用方法。

2. #### 方法重载  
方法重载=***同一个类中方法名相同而形参不同(个数、类型、顺序)***  
在调用方法时，会寻找相同的方法名下，形参能与实参相对应的那个方法对应并运行。

3. #### 参数数据类型转换
实参数据向形参传输时，系统先优先匹配能够完美对应的方法；若不能完美对应，则会自动将部分数据由小及大(不能由大及小)的类型转换，然后再匹配能够对应的方法；受方法重载的影响，一些数据类型转换可能会对应同等级的几个方法，此时系统会无法选择而报错  
### 存储原理：
1. java运行机制：  
  .java源文件->编译生成.class字节码->JVM运行(java虚拟机)
  也就是java程序不在操作系统中运行，通过虚拟机可以方便程序跨操作系统运行。
2. java内存分配：  
  - #### 虚拟机栈：java方法运行的内存空间，随一个方法调用而添加，结束而销毁
  - #### 堆：所有对象和数组的内存池，new创建的对象实例和数组等会把地址设置在堆中
  - #### 方法区：存储类结构、常量、静态变量等元数据的区域，例如即时编译器编译后的代码
  - 程序计数器：记录当前线程执行到哪一行代码
  - 本地方法栈：为native方法提供的栈空间
3. #### 数组的内存和传递  
   java除了八大基本数据类型，三大引用数据类型(类、接口、数组)都是引用数据类型，它们存储的内容是地址，对象本体在堆，赋值实际上就是在复制地址。

   在向方法传递基本数据类型时，是往形参里传递一个真实的数据副本，形参的改变不会影响实参，java中对基本数据类型没有直接的“指针”角色。

   在向方法传递引用数据类型时，是往形参里传递一个地址副本，通过使用形参能够改变地址上的数据，也就是形参的改变会影响实参(注意这是数据本身的地址，不是形参实参的地址)。

# 3.16 面向对象基础
1. 类和对象  

类就是对象共同特征的描述；  
对象就是真实存在的具体东西。  
面向对象就是通过事先制定好的类划分对象，在解决问题时不用管操作的过程，通过类处理问题。

定义一个类：
```
public class 类名{
  1. 成员变量(代表属性)，定义类的各个属性名称,一般不初始化
  2. 成员方法(代表行为)，定义类封装的内部方法
  3. 构造器
  4. 代码块
  5. 内部类
}
```
public是Java中的访问修饰符,被它修饰的类完全公开,能被所有其他类访问。一个java文件中能同时定义多个类，但public class类名必须和文件名一致，且一个源文件最多一个public class。  

***定义一个类，实际上就是自己定义了一个引用数据类型***  

定义一个类的对象：
```
类名 对象名 = new 类名();
```
访问对象的操作：
```
访问属性：对象名.属性名
访问行为：对象名.方法名(实参1....)
```
2. #### 封装
封装：a. 类能描述对象，储存对象的数据，也应包含对象(状态)处理的各种方法 b.将类的属性（数据）私有化（用 private 修饰），同时提供公共的方法（get/set） 对外暴露访问入口，隐藏内部实现细节，只开放有限的、可控的交互方式，防止数据错误或不合法。

#### private修饰符：可以修饰类的成员(属性方法等)，被修饰的成员只能在当前类中直接访问。

经private修饰后的属性成员，只能通过在类中定义新的方法，专门提供对外接口方便外界修改或获取数据，能够自定义检查过程筛选，能极大的提高代码数据安全性，注意这样的接口方法应该用public修饰。

3. #### 就近原则
局部变量：定义在方法内的变量，存储在堆内存中  
成员变量：定义在类内，在方法外的变量，存储在栈内存中  

#### 一对局部变量和成员变量同名，在调用时，所采取的数据是从上到下离当前行代码最近的定义，如果想略过局部变量使用成员变量，应该在调用变量时，在变量前加上“this.”

从内存的角度解释，类所定义的对象，是个引用数据类型，本质存储了一个堆内存的地址。在通过对象调用它的方法时，this.会记录调用者的储存值，也就是堆内存的地址，所以能够从地址上把局部变量和成员变量。

4. #### 构造方法

构造方法也就是构造器或者构造函数，在创建对象时，虚拟机自动调用构造方法给成员变量初始化。

定义格式(在类中)：
```
修饰符 类名(成员变量形参){
  方法体；
}
```
a.  修饰符用 private 构造方法禁止外部创建对象，暂不叙述；一般的用public 构造方法。  
b. 方法的名与当前的类名相同  
c. 方法没有具体的返回值类型(并不是void)  
d. 每次创建对象，虚拟机就调用构造方法一次  
e. 如果没有书写构造方法，虚拟机自动形成无参数的构造方法，成员变量赋予默认值  
f. 书写构造方法后，虚拟机不在自动生成无参构造，一般需要手动书写有参和无参构造，这样的方法也符合重载规律。  
g. 注意有参构造的成员变量赋值，this.的运用。  
h. 应对有参构造，定义对象时类名括号里就应该写对应实参  

5. #### 标准javaBean类
满足以下条件：  
a. 类名易理解，用public修饰  
b. 成员变量要用private修饰  
c. 每个成员变量要有自己的读改操作方法  
d. 至少要书写无参和全参两种构造方法

6. #### 对象内存运行原理
一个对象的内存运行过程：  
1. 把主.class文件和自定义类.class文件加载到方法区(包含里面的成员变量和方法),main方法入栈内存
2. 在使用自定义类声明局部变量时，这是定义了一个指向该类对象的地址值变量，此时还是空白引用
3. 堆内存开辟空间：new 关键字触发 JVM 在堆内存中分配一块连续的内存空间，用于存放 Student 对象的成员变量，以及成员方法的地址引用。
4. 默认初始化：JVM 自动为堆中的成员变量赋默认初始值，引用类型（如 String name）默认值为null，基本类型（如 int age）默认值为0，保证对象创建后不会因无初始值报错。
5. 显示初始化：若代码中给成员变量手动设置了初始值（如String name = "张三";），这一步会用手动赋值覆盖默认值
6. 构造方法初始化：执行 Student 类的构造方法，若构造方法内有赋值逻辑，会在这一步完成最终的对象初始化。
7. 地址赋值给局部变量：将堆中创建的对象的内存地址赋值给栈中的局部变量，此时局部变量就持有了对象的引用，可通过对象访问堆中的成员变量和方法。
